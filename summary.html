기본개념 네트워크란?
네트워크 = 컴퓨터나 다양한 디지털 장치들이 유선·무선 중계기인 라우터와 스위치를 통해 연결되어, 각각 고유의 네트워크 주소로 서로를 구분하고 데이터를 송수신하는 통신망
⭐ 인터넷의 시초: 1969년 최초 컴퓨터 통신 테스트 → 'LOGIN' 보내려다 'L', 'O' 2글자만 전송 성공
핵심 장치 비교
장치역할특징
라우터(router)서로 다른 네트워크 연결 + 가장 효율적인 경로 안내게이트웨이 기능 포함. 가정용 = 소형(유무선 공유기)
스위치(switch)네트워크 내에서 장치들 연결 + 패킷 송수신소규모 네트워크, 대용량 패킷 처리
랜 카드외부 네트워크와 최초 접속컴퓨터 본체 확장 슬롯에 삽입
AP(Access Point)유선을 무선으로 전환라우팅 기능 없이 무선 공유만 필요할 때

네트워크 구조 유무선 & 네트워크 종류
유선 vs 무선
📡 유선 네트워크
통신 매체: UTP 케이블, 광섬유 케이블
장점
  • 신뢰도 높음
  • 속도 빠름
단점
  • 이동 어려움
  • 모든 컴퓨터를 선으로 직접 연결 → 회선 복잡
📶 무선 네트워크
통신 매체: NFC, 와이파이, 블루투스, RFID
장점
  • 이동 자유로움
  • 물리적 연결 불필요
  • 다수 동시 이용 가능
단점
  • 보안 취약
  • 신호 감쇄 심함
  • 유선보다 속도 느릴 수 있음
  • 전송 범위 벗어나면 연결 어려움
네트워크 종류 (범위 순서)
종류범위설명예시
PAN
Personal Area Network		수 m아주 작은 영역무선 키보드, 마우스, 블루투스, 에어드랍
LAN
Local Area Network		수m ~ 수km건물 내부 유무선 혼합 네트워크. 가장 많이 쓰는 서브넷교실, 회사 건물
MAN
Metropolitan Area Network		수km ~ 수십km도시 내 네트워크 (LAN 여러 개 모임)도시 통신망
WAN
Wide Area Network		수십km 이상도시 벗어난 범위 (MAN 여러 개 모임)인터넷
⭐ 인터넷 접속 = MAN 범위를 넘어 WAN에 접속하는 것
서브넷(subnet) = IP 주소에서 네트워크 영역을 부분적으로 나눈 부분망. 예) 가정, 학교 각 반
패킷 데이터 전송 방식
패킷(packet) = 네트워크를 통해 전송하기 쉽도록 데이터를 적당한 크기로 자른 최소 전송 단위
특징설명
분할 전송통신 매체 용량 제한 + 네트워크 혼잡도에 따라 패킷마다 다른 경로로 전송
패킷 내용수신자 주소 + 다양한 데이터 포함
수신 후 처리수신자가 주소 제외한 데이터만 조합해 재구성
단위 크기 예시사진 전송 시 1,500byte씩 분할
네트워크 설정 DHCP & 주요 용어
DHCP(Dynamic Host Configuration Protocol) = IP 주소를 미리 지정하지 않고 DHCP 서버가 네트워크 접속 시 자동 할당
DHCP 사용 이유: ① IP 충돌 방지 ② 높은 이동성 보장 ③ 수동 입력 번거로움 없이 효율적 관리
용어설명예시

서브넷 마스크IP 주소를 네트워크 주소 부분과 호스트 주소 부분으로 분리255.255.255.0
게이트웨이외부 네트워크를 연결해 주는 장치 주소 (보통 라우터 주소)192.168.1.1
DNS 서버도메인 이름 → IP 주소 변환EBS.co.kr → 110.165.22.169

개념 사물 인터넷(IoT)이란?
사물 인터넷(IoT: Internet of Things) = 사물에 다양한 센서와 통신 기능을 장착하여 사물 간에 정보 교류가 가능하게 만든 것. 각각의 사물이 제공하던 것 이상의 새로운 가치를 제공하는 기술
⭐ 여기서 '사물' = 네트워크에 연결될 수 있는 모든 것 (자동차, 집, 가전제품, 스마트폰, 옷, 신발 등)
⭐ 사람이 직접 명령을 내리지 않아도 사물끼리 서로 정보를 주고받으며 스스로 작동

대표 예시: 하이패스 통행료 처리 시스템 — 달리는 차 안에서 자동 결제

사용자 제공
D — 디바이스(Device)
센서 + 액추에이터 + 통신 모듈 + 마이크로컨트롤러로 구성된 장치
마이크로컨트롤러 = 소형 CPU와 입출력 모듈을 하나의 칩으로 구성한 작은 컴퓨터 (예: 라즈베리파이 피코, ESP32, 아두이노)
센서/액추에이터기능활용 예
조도 센서빛의 밝기 측정블라인드 열고 닫기
인체 감지 센서사람의 움직임 감지전등 켜고 끄기
토양 수분 센서토양의 수분 측정식물 관리
온습도 센서온도·습도 자동 측정에어컨 제어, 냉장고 온도 유지
초음파 센서거리 측정자동차 근접 물체 감지
가스 센서공기 중 유해 성분 감지공기 청정기 자동 작동
이미지 센서카메라 이미지 정보 획득냉장고 내부 재료 파악
모터속도·회전 방향 제어선풍기, 무선 조종 자동차
N — 네트워크(Network)
센서가 수집한 데이터를 전송하거나 사물들이 서로 정보를 교환하도록 연결해 주는 매개체
기술특징활용 예
RFID무선 주파수로 사물 식별재고 관리, 하이패스
NFC근거리 무선 통신금융 결제, 교통 카드, 스마트 홈
ZigBee적은 소비 전력으로 근거리 무선 통신센서 네트워크, 스마트 홈·빌딩
Beacon(비콘)블루투스 기반 근거리 데이터 전송박물관, 전시회
Wi-Fi전자 기기 무선 통신일반 인터넷 연결
블루투스근거리 무선 통신스마트폰 주변 기기 연결
P — 플랫폼(Platform)
다수의 사용자와 사물들 사이에 존재하여 중계자 역할을 하는 시스템
디바이스 데이터를 클라우드에 저장·관리·분석 → 사용자가 언제 어디서나 사물 상태 확인 가능
C/S — 서비스(Contents/Service)
센서로 수집한 정보를 플랫폼에서 융합하여 사용자에게 맞춤형 서비스 제공
적용 분야: 헬스케어, 스마트 가전, 의류, 쇼핑, 자동차, 농축수산업, 금융, 도시, 제조업 등
서비스 예시설명
스마트 가전스마트폰/태블릿으로 원격 제어·모니터링
스마트 의류센서로 호흡, 체온 등 생체 신호 측정
스마트 결제스마트폰에 결제 정보 등록 → 비밀번호/지문 인식으로 결제
스마트 쇼핑위치·정보 확인 → 터치 동작으로 상품 검색 및 자동 결제
스마트 자동 사료 급식카메라+이미지 인식으로 사료 섭취량 파악

개념 디지털 데이터 & 압축 개요
정형 데이터
미리 정해 놓은 형태. 행·열 테이블로 저장 가능. 예) 날짜, 이름, 주소
비정형 데이터
형식이 정해지지 않은 데이터. 예) 문자, 소리, 이미지, 동영상
데이터 압축
원본 데이터를 더 작은 용량의 데이터로 만드는 것
⭐ 압축 필요성: ① 저장 공간 절약 ② 전송 시간 단축 ③ 비용 절감 ④ 전기 사용 감소(환경) ⑤ 데이터 보안 강화
핵심비교 무손실 압축 vs 손실 압축
구분무손실 압축손실 압축
데이터 손실없음있음
원본과 비교거의 같음다를 수 있음
압축률낮음높음
용량큰 편작은 편
파일 형식ZIP, GIF, PNGJPEG, MPEG, MP3
적합한 데이터텍스트, 은행 기록 (손실 시 문제)사진, 영상, 음악
⭐ 암기 팁 — 무손실: ZIP/GIF/PNG (Z·G·P = 잃지 않음) | 손실: JPEG/MPEG/MP3 (J·M·M = 줄여도 됨)
압축 기법 런 길이 부호화 (Run Length Encoding)
런 길이 부호화 = 동일한 데이터가 몇 번 반복되는지 기록하여 데이터를 짧게 만드는 방식
대표 파일 형식: BMP
압축 전압축 후원본 크기압축 크기압축률
AAADDBOOOOA3D2B1O410 bytes8 bytes10/8 = 1.25
HAAAAAAPPYH1A6P2Y110 bytes8 bytes1.25
⚠️ 한계: 연속적인 반복이 많은 데이터에만 효율적. 아이콘·로고(단색) → 효율적 / 다양한 색 사진 → 부적합
압축률 = 변환 전 데이터 크기 ÷ 변환 후 데이터 크기. 높을수록 압축이 잘 된 것
암호화 데이터 암호화 (정보보안 연결)
암호화 = 정보를 타인이 알아볼 수 없는 암호문 형태로 저장·전송하는 방법
→ 손실되더라도 사용자의 비밀 보장 가능
변수·자료형 변수와 자료형
변수 = 어떤 값을 저장하고 나중에 참조하기 위한 기억 장소
대입 연산자(=) = '같다'가 아니라 '우측의 데이터를 좌측 변수에 저장'하라는 의미
예) x = x + 1 → 변수 x의 값에 1을 더한 후 다시 x에 저장
자료형 종류
구분자료형표현 방법특징

튜플 tuple(10, 20, 30)괄호, 순서 있음, 수정 불가

  • 괄호 안 내용을 화면에 출력
  • 여러 값 → 콤마(,)로 구분
  • 출력 후 자동 줄바꿈
  • 수식 넣으면 연산 결과 출력
표준 입력 input()
  • 키보드로 입력받은 값을 문자열로 저장
  • 괄호 안 문자열을 화면에 표시
  • 정수로 변환 필요시 int() 사용
# print 예시
fruit = '청포도'
print('내가 좋아하는 과일은', fruit, '입니다.')
print(7, '*', 5, '의 계산 결과는', 7*5, '입니다.')

# input 예시 (결과는 문자열!)
age = input('몇 살이세요?')
print('당신은', age, '살이군요.')
파일입출력
파일 변수 = open(파일명, 파일입출력 모드) → 처리 → close()
모드설명함께 쓰는 함수
'r' 읽기파일 데이터 읽음 (파일 없으면 오류)read(), readline(), readlines()
'w' 쓰기파일에 데이터 씀 (있으면 기존 삭제, 없으면 생성)write()
'a' 추가파일에 데이터 이어씀 (있던 데이터 유지)write()
# 파일 쓰기
f = open('파일명.txt', 'w')
f.write('내용\n')
f.close()

# 파일 읽기
f = open('파일명.txt', 'r')
a = f.read()
print(a)
f.close()
⭐ read(): 전체 반환 | readline(): 한 줄씩 | readlines(): 모든 줄을 리스트로 반환
⭐ with 명령어 사용 시 close() 자동 처리
다차원 데이터 리스트 & 딕셔너리
차원설명예시
1차원여러 개의 값이 모인 것[95, 85, 100, 95]
2차원여러 개의 1차원 데이터가 모인 것[[95,250,'M'],[85,230,'S']]
3차원여러 개의 2차원 데이터가 모인 것반별 학생 데이터 묶음
리스트 핵심
# 리스트: 인덱스 0부터 시작
T_size = [95, 85, 100, 95]
print(T_size[0])      # 95 (1번 학생)
T_size[0] = 100     # 수정
T_size.append(90)   # 마지막에 추가 (한 번에 한 개만)

# 2차원 리스트: [행번호][열번호]
class1 = [[95, 250, 'M'],
          [85, 230, 'S']]
print(class1[0])      # [95, 250, 'M'] (0행 전체)
print(class1[1][0])  # 85 (1행 0열)

# 딕셔너리: 키로 접근

print(T_size['내빈'])       # 95
T_size['내빈'] = 100        # 수정
T_size['재우'] = 105        # 추가 (없는 키면 새로 생성)

# 2차원 딕셔너리: 딕셔너리[키][키]

print(class1['내빈']['상의'])  # 95
제어구조 순차 · 선택 · 반복
제어 구조설명키워드
순차 구조위에서 아래로 순서대로 실행 (기본 구조)-
선택 구조조건이 참/거짓에 따라 다른 명령 실행if / if~else / if~elif~else
반복 구조조건 만족하는 동안 명령 반복 실행for / while
선택 구조
# if문 — 조건이 참일 때만 실행
if bus == 'y':
    print('문이 열립니다.')
print('다음 정거장은 시청역.')  # 항상 실행

# if~else문 — 참이면 명령1, 거짓이면 명령2
if number % 2 == 0:
    print(number, '는 짝수.')
else:
    print(number, '는 홀수.')

# if~elif~else문 — 여러 조건
if number > 0:
    print('양수')
elif number < 0:
    print('음수')
else:
    print('0')
⭐ 비교 연산자: == (같음) / != (다름) / > / >= / < / <=
⭐ 논리 연산자: and (둘 다 참) / or (하나라도 참) / not (반전)
⭐ % 연산자: 나눗셈 나머지 (짝수 판단: n%2==0)
반복 구조
# for문 — 반복 범위만큼 실행
for i in ['사과', '수박', '참외']:
    print(i, '는 맛있다.')

# range() 함수 — for문과 자주 사용
# range(끝): 0부터 끝-1까지
# range(시작, 끝): 시작부터 끝-1까지
# range(시작, 끝, 간격): 간격만큼 증가
for i in range(1, 10):
    print(f'5 X {i} = {i*5}')  # f-string
⭐ range() 예시: range(5) → 0,1,2,3,4 | range(5,10) → 5,6,7,8,9 | range(0,10,2) → 0,2,4,6,8 | range(5,0,-1) → 5,4,3,2,1

지능 에이전트 에이전트 vs 지능 에이전트
에이전트 (Agent)
  • 환경을 인식하고 정해진 규칙에 따라 행동
  • 복잡한 추론·학습 없이 목표 달성
  • 스스로 반응하며 자동 행동
예) 온도 조절기, 자동문
지능 에이전트 (Intelligent Agent)
  • 인공지능으로 스스로 환경 인식·목표 설정
  • 학습과 추론을 통해 합리적 행동 결정
  • 환경 변화에 적응, 행동 지속적으로 개선
예) 추천 시스템, 자율주행 자동차
⭐ 에이전트: 규칙 기반 단순 행동 | 지능 에이전트: 학습+추론으로 복잡한 작업도 능동적 수행
⭐ 지능 = 학습과 추론을 통해 문제를 해결하고 적절한 결정을 내리는 과정
환경(environment) = 에이전트가 상호 작용하고 인식하며 행동을 수행하기 위한 모든 외부 조건
기계학습 기계학습이란?
기계학습 = 컴퓨터가 데이터를 기반으로 스스로 학습하여 규칙을 찾고, 그 규칙으로 새로운 데이터 결과를 예측하는 것
사람: 경험을 토대로 예측 | 기계학습: 데이터를 사용하여 예측
전통적 프로그래밍 vs 기계학습
전통적 프로그래밍
  • 사람이 직접 규칙·조건 정의
  • 입력 + 규칙 → 정답 출력
  • 새로운 상황 생길 때마다 규칙 수정 필요
예) 이메일 제목에 '이벤트' 단어 → 스팸으로 분류
기계학습
  • 컴퓨터가 데이터로 스스로 규칙 학습
  • 입력 + 정답(레이블) → 규칙 도출
  • 새로운 문제에도 규칙 새로 만들 필요 없음
예) 수많은 이메일 학습 → 스팸 패턴 스스로 파악
⭐ 레이블 = 각 입력 데이터에 대해 원하는 출력값(정답) 또는 목표값
학습 유형 지도학습 · 비지도학습 · 강화학습
유형특징방식활용 예
지도학습
(Supervised)		 입력 데이터 + 레이블 함께 학습 새로운 데이터 → 예측값 출력 강아지/고양이 분류, 질병 유무 판단, 스팸 메일 분류, 날씨 예측
비지도학습
(Unsupervised)		 레이블 없는 데이터 → 숨은 패턴·의도 찾기 비슷한 특성끼리 그룹(군집)으로 묶음 고객 구매 패턴 분석, 상품 추천, 유사 그룹 발견
강화학습
(Reinforcement)		 변화하는 환경에서 에이전트가 학습 행동(action) 중 최대 보상(reward)을 주는 행동 학습 게임 AI, 자율주행
⭐ 지도학습: 레이블 있음 / 비지도학습: 레이블 없음 / 강화학습: 보상으로 학습
⭐ 지도·비지도 = 정적 환경 | 강화학습 = 변화하는 환경
개념 정보 보안이란?
정보 보안 = 정보의 생성, 처리, 저장, 전송 등의 전 과정에서 허가되지 않은 접근·사용·공개·중단·수정·파괴로부터 정보를 안전하게 유지하는 방법과 절차
